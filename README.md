# PyDopplerSim

PyDopplerSim is a Python baseband IQ simulator for mobile transmitter (TX) and
receiver (RX) Doppler scenarios. It generates complex IQ, estimates Doppler
from the received signal, and compares RX-observable reconstruction against
simulator ground truth.

The project is useful for thesis work and engineering experiments where the
important question is not only "what is the true geometry?", but "what can the
RX reconstruct from the signal it actually has?"

## RX Observability Boundary

The RX-minimal reconstruction path is intentionally narrow:

- From Doppler shift alone, RX can recover range rate:

```text
r_dot = -delta_f * c / fc
```

- RX can estimate range acceleration by differentiating a decimated/smoothed
  `r_dot` series:

```text
r_ddot = d(r_dot) / dt
```

- Optional RX velocity knowledge adds velocity-aware constraints, but it does
  not make range, bearing, or TX position observable.

Simulator plots may show TX path, RX path, range, and bearing as diagnostic
ground truth. Those overlays are labeled because they are not RX-minimal
reconstruction outputs.

## Coordinate System And Scenarios

- `x`: longitudinal direction of travel.
- `y`: lateral lane offset.
- RX travels along `+x` in the built-in scenarios.
- Positive `r_dot` means TX and RX are moving apart.
- Negative `r_dot` means they are approaching.

Built-in scenarios:

| Scenario | Description |
| --- | --- |
| `colocated` | TX and RX move together with a small lateral offset for visibility. |
| `same-direction` | TX is ahead in an adjacent lane and RX overtakes. |
| `oncoming` | TX approaches from the opposite direction. |

## Signal Model

The simulator generates a continuous-wave complex baseband signal:

```text
iq(t) = exp(j * 2*pi * integral(f_tone + delta_f(t)) dt) + noise
```

`delta_f(t)` comes from the simulated geometry:

```text
delta_f = -r_dot * fc / c
```

Noise is complex AWGN controlled by `SNR_DB`.

## Estimation And Endpoint Masking

The phase-difference estimator uses the lag-1 complex phase increment:

```text
delta_f[n] = angle(iq[n] * conj(iq[n-1])) * fs / (2*pi)
```

That estimate is smoothed with a Hann window. The first and last
`PD_SMOOTH_WINDOW // 2` samples are filter-edge transients. They can appear as
large IF/Doppler samples in plots, especially when true Doppler is near zero
and noise dominates the lag-1 phase estimate.

Those endpoint samples are masked in the IF/Doppler plots. They are estimator
edge artifacts, not physical Doppler.

## Outputs

Running a scenario writes outputs under `plots/{scenario}/`.

| File | Meaning |
| --- | --- |
| `trajectory.png` | Ground-truth TX/RX paths and range diagnostic. |
| `doppler.png` | Phase-diff Doppler estimate, STFT peak, and ground-truth diagnostic overlay. |
| `rdot_rddot.png` | RX-recovered `r_dot` and `r_ddot` compared with ground truth. |
| `animation.mp4` / `animation.gif` | Positions-only animation. |
| `animation_spectrogram.mp4` / `animation_spectrogram.gif` | Positions plus live STFT spectrogram animation. |

`tx_derivation.png` is deprecated and is no longer generated by default. The
old function remains callable for compatibility, but TX path derivation is not
part of the RX-minimal reconstruction model.

## Installation

```bash
pip install -e ".[dev]"
```

## Running

Run every scenario:

```bash
python main.py
```

Run one scenario:

```bash
python main.py --scenario oncoming
```

Choose output animation file formats:

```bash
python main.py --render-formats gif
python main.py --render-formats mp4
python main.py --render-formats none
```

Animation variants are configured in `config.py`:

```python
class SimConfig:
    ANIMATION_VARIANTS = "both"  # "positions", "spectrogram", or "both"
```

## Configuration

Important `SimConfig` values:

| Setting | Meaning |
| --- | --- |
| `FC` | Carrier frequency. |
| `FS` | IQ sample rate. |
| `SNR_DB` | Signal-to-noise ratio. |
| `RX_VX` | RX longitudinal velocity. |
| `RX_KNOWS_VELOCITY` | Allows RX reconstruction to use its own velocity. |
| `PD_SMOOTH_WINDOW` | Phase-diff smoother length. |
| `RDOT_DECIM_HZ` | Decimation rate before differentiating `r_dot`. |
| `RENDER_FORMATS` | `"mp4"`, `"gif"`, `"both"`, or `"none"`. |
| `ANIMATION_VARIANTS` | `"positions"`, `"spectrogram"`, or `"both"`. |

## Custom TX Paths

Use `WaypointPath` or the path helpers in `paths.py`:

```python
from config import scenario_oncoming
from iq_gen import generate_iq
from paths import angled_path

cfg = scenario_oncoming()
path = angled_path(
    x0=cfg.tx_x0,
    y0=cfg.tx_y,
    angle_deg=10.0,
    speed=cfg.tx_vx,
    duration=cfg.duration,
)

result = generate_iq(cfg, path_provider=path)
```

## Tests

Run fast tests:

```bash
python -m pytest tests -m "not slow" -q
```

Run focused estimator tests:

```bash
python -m pytest tests/test_estimation.py tests/test_estimation_properties.py -q
```

Run all tests, including slow integration cases:

```bash
python -m pytest tests
```

Unit fixtures use smaller sample rates and durations so fast tests complete
quickly. Full-rate/default-duration simulations belong in slow integration
tests or manual scenario runs.

## Known Limitations

- Range, bearing, and TX position are not observable from Doppler alone.
- RX velocity knowledge does not resolve range, bearing, or TX position.
- Ground-truth overlays are simulator diagnostics and are labeled as such.
- Animation MP4 output requires ffmpeg. Set `FFMPEG_PATH` in `config.py` if it
  is not available on `PATH`.
- `tx_derivation` is deprecated and excluded from default outputs.
